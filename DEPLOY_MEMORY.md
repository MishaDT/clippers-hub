# Памятка для будущего деплоя

Важно: перед Netlify сначала Миша смотрит проект локально на `http://localhost:3000`, мы правим UX/UI, и только после подтверждения готовим выгрузку на Netlify.

Когда будем выкладывать проект на Netlify, обязательно учесть:

1. Локально сейчас используется SQLite, но для нормального сайта нужна облачная PostgreSQL-база.
2. Рекомендуемый вариант: Netlify + Supabase + GitHub.
3. Перед деплоем переключить `prisma/schema.prisma` на PostgreSQL.
4. В Netlify добавить переменные:
   - `DATABASE_URL`
   - `DIRECT_URL`
   - `SESSION_SECRET`
   - платежные ключи, когда будем подключать реальные оплаты
5. После подключения базы выполнить миграции и seed-данные.
6. Не вводить реальные карты, пока платежи работают в демо-режиме.
7. Для продакшена заменить тестовые данные, демо-ссылки и мок-синхронизацию просмотров на реальные API.

Текущая цель разработки перед деплоем:

- сделать сайт похожим на живое мобильное приложение;
- добавить 100+ клипов/заказов/активностей;
- сделать нижнее меню как в соцсетях, с иконками;
- упростить UX, убрать лишнюю гиковость;
- сделать дизайн спокойнее, дороже и понятнее.
